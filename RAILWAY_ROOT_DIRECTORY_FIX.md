# CRITICAL: Fix Railway Root Directory

## The Problem

Railway is scanning your ENTIRE repository (including the frontend with Next.js) instead of just the backend. That's why you're seeing Next.js security vulnerabilities even though you're deploying Django.

## The Solution

You MUST set the Root Directory in Railway to tell it to ONLY look at the backend folder.

---

## Step-by-Step Instructions

### 1. Go to Railway Dashboard

Open: https://railway.app/dashboard

### 2. Click on Your Project

Click on the **keja** project

### 3. Click on the Service

Click on the **keja-backend** service (or whatever you named it)

### 4. Go to Settings Tab

Click on the **Settings** tab at the top

### 5. Find "Source" Section

Scroll down until you see a section called **"Source"** or **"Build"**

### 6. Set Root Directory

Look for a field called:
- **Root Directory** OR
- **Source Directory** OR  
- **Watch Paths**

Enter: `keja/apps/backend`

### 7. Save Changes

Click **Save** or **Update**

### 8. Redeploy

Click **Deploy** button or it will auto-deploy

---

## Alternative: Deploy from Subdirectory

If you can't find the Root Directory setting, try this:

### Option A: Use Railway CLI

```bash
# Install Railway CLI
npm install -g @railway/cli

# Login
railway login

# Link to your project
railway link

# Deploy from backend directory
cd keja/apps/backend
railway up
```

### Option B: Create New Service

1. In Railway, click **"+ New"**
2. Select **"GitHub Repo"**
3. Choose your repository
4. **IMPORTANT:** When it asks for the directory, enter: `keja/apps/backend`
5. Add PostgreSQL database
6. Set environment variables
7. Deploy

---

## Verify Root Directory is Set

After setting the root directory, check the deployment logs. You should see:

✅ **Good (Backend only):**
```
Installing Python dependencies...
Collecting Django==5.2.7
Collecting djangorestframework==3.16.1
...
```

❌ **Bad (Still scanning frontend):**
```
Found 1 vulnerable package(s):
next@15.2.8
```

---

## If It Still Fails

### Disable Security Scanning Temporarily

1. Go to Railway Project Settings
2. Find **Security** section
3. Toggle **"Vulnerability Scanning"** to OFF
4. Deploy again
5. Re-enable after successful deployment

### Or: Separate the Repositories

If Railway keeps scanning the entire repo, consider:

1. Create a new repo: `keja-backend`
2. Copy only the `keja/apps/backend` folder
3. Deploy from that separate repo

## Environment Variables Needed

Make sure these are set in Railway:

```bash
DJANGO_SECRET_KEY=<generate-new-key>
DEBUG=False
ALLOWED_HOSTS=your-app.railway.app
DJANGO_SETTINGS_MODULE=keja_backend.settings_production
DATABASE_URL=<auto-generated-by-railway>
CORS_ALLOWED_ORIGINS=https://your-frontend.vercel.app
```
